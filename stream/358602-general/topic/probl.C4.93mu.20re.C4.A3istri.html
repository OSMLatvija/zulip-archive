Piemēri: nepareizas ceļazīmes (LVC ātruma ierobežojumi, pašvaldību dzīvojamo zonu sākums beigas, īpaši šķības ēkas VZD datos utt).

Funkcionāli derētu:

  • Ļoti viegla pievienošana un rediģēšana.
  • Teksta un vizuālais materiāls (piem, JOSM screenshoti, bildes dabā utt), linki (OSM, Mapillary utt).
  • Potenciāli iespēja rediģēt "lietotājiem no malas", ja nu varbūt VZD vai kāds cits vēlas iesaistīties kādā brīdī (tīri web interfeiss).
  • Iespēja viegli atvērt to visu JOSM, uMap vai tamlīdzīgā rīkā.
  • Ērtas kategorijas, meklēšana, filtrēšana.

Iespējamie varianti:

  • Google Sheets spreadsheets
  • Kaut kas Github?
  • Tabula OSM wiki

Kādas vēl idejas, vēlmes, ieteikumi?

Mums šeit ir iespēja taisīt tā saucamus "code playgrounds", kas saistīti ar overpass-turbo.eu:

[out:json][timeout:25];
// fetch area “Latvia” to search in
{{geocodeArea:Latvia}}->.searchArea;
// gather results
(
  // query part for: “building=hangar”
  node["building"="hangar"](area.searchArea);
  way["building"="hangar"](area.searchArea);
  relation["building"="hangar"](area.searchArea);
);
// print results
out body;
>;
out skel qt;

Desktop un web versijās parādās podziņa, lai atvērt to koda gabalu caur overpass-turbo.eu, cerams ka mobilai aplikācijai arī pievienos šo fīču.

Vienkārši jāuztaisa koda bloks ar overpass kā valodu:

```overpass
// šeit jābūt Overpass query
```

Tas šķiet labāk no arhivēšanas skatpunkta (nezinu, cik ilgi tie Overpass linki darbojās) un arī foršāk uzreiz redzēt Overpass kodu čatā, lai saprastu, par ko ir runa.
